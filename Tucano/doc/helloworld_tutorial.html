Hello World

If you were able to compile and run the helloworld example in the previous Compiling and Running Tutorials, you are all set to start. Here we will guide you step-by-step through the hello world application. Even though this is a very simple application, the main structure will be the same for the following exercises and assignments.

In the helloword/src folder there are four files:

  • main.cpp
  • widget.hpp
  • widget.cpp
  • helloworld.hpp

main.cpp

The main.cpp file contains mostly the window setup using GLFW. In the main function, a GLFW window is created. This is also where the main display loop is. It will render the widget every frame.

The other important function is initialize. Here Tucano will first initialize GLEW. It will then output some information about your driver and OpenGL and GLSL versions.

widget.ccp and widget.hpp

The widget holds all elements for rendering, such as meshes, cameras, lights, GUI, and effects. Note that a widget can have multiple effects and switch between if necessary. For this simple example it has only three methods.

  • initialize: initializes your scene (camera, shaders, and OpenGL variables)
  • createGeometry: creates a single triangle to be rendered
  • render: calls the class responsible for rendering the scene every frame

helloworld.hpp

The helloworld is an Effect, following Tucano's terminology. It means that it receives your scene (camera, model, light, etc ...) and renders it using some effect. It might be a very simple effect, such as coloring all vertices with a default color (as in this example), or more sophisticated shading effects that you will learn later on.
